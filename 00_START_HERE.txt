📊 ARCHITECTURE
================================================================================

ROOM-BASED SYSTEM:
  Each tracking session has a unique room (e.g., "A7K9F2")
  • Device joins room and sends position updates
  • Owner(s) join same room and receive updates
  • Server relays all events between participants
  • Room deleted when all clients disconnect

REAL-TIME EVENT FLOW:
  Device → position:update → Server → position_update → Owner
  Owner → device:ring → Server → device:ring → Device → Sound Alert

MESSAGE LATENCY:
  Device → Server: ~50-100ms
  Server → Owner: ~50-100ms
  Total round-trip: ~100-200ms (well under 3-second requirement)
